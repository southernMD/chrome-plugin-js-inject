JS Script Manager

使用说明

这个扩展用于管理本地 JS 脚本，并在当前页面直接执行。

首次使用

  1. 在 chrome://extensions 中加载 Z:\cargo-admin\chrome-plugin。
  2. 确保 Native Host 已注册，否则导入、删除、打开本地文件这些功能无法使用。
  3. 打开扩展弹窗后，点击 导入脚本 选择一个本地 .js 文件。

Native Host 注册方式

  1. 打开 chrome://extensions，找到这个扩展并复制扩展 ID。
  2. 进入扩展目录下的 native-host 文件夹。
  3. 打开 PowerShell，执行下面这条命令：

powershell -ExecutionPolicy Bypass -File .\register-host.ps1 -ExtensionId "你的扩展ID"

例子：先在文件资源管理器打开 chrome-plugin\native-host，然后在该目录的 PowerShell 里直接运行上面的命令。

  1. 命令执行成功后，刷新扩展。
  2. 之后扩展里的 导入脚本、删除、打开本地文件 就都能用了。

按钮说明

  • 导入脚本：把你选择的 .js 复制到 chrome-plugin/scripts，并自动更新清单。
  • 加载刷新：重新读取脚本清单并刷新列表。
  • 在当前页面执行：在当前标签页主世界执行该脚本。
  • 打开本地文件：在你的电脑本地用默认程序打开这个脚本文件。
  • 删除：真实删除 chrome-plugin/scripts 里的对应脚本文件，并更新清单。

标题编辑

卡片标题可以直接编辑，这个标题只保存在扩展本地存储里，不会修改真实文件名，也不会改 index.json。

常见问题

  • 如果导入、删除、打开本地文件失败，优先检查 Native Host 是否注册成功。
  • 如果图标、按钮或逻辑更新没生效，先刷新扩展。
  • 如果执行脚本提示找不到实例，说明页面上下文或脚本内容本身还需要调整。
